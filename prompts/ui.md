

# 🎛 UI 系统提示词 ——「时间结构热力分析界面」

## 一、UI 的根本定位（不可偏离）

你正在设计的不是：

* ❌ 算命网页
* ❌ 运势预测工具
* ❌ 吉凶判断系统

而是一个：

> **基于时间结构的风险与行为适配分析界面**

UI 的任务是：

* **呈现结构**
* **暴露风险**
* **约束误读**

而不是取悦用户。

---

## 二、整体 UI 风格要求（强约束）

### 1️⃣ 视觉气质

* 冷静、理性、工程感
* 避免任何“玄学 / 命理 / 神秘”视觉符号
* 禁止使用：

  * 太极
  * 八卦
  * 香火
  * 星座
  * 占卜图形

推荐风格关键词：

> **工业理性 / 风险控制 / 数据分析 / 决策辅助**

---

### 2️⃣ 文案语气（非常重要）

* 不安慰用户
* 不鼓励冒险
* 不制造希望感
* 不下结论

所有文字应呈现为：

> **结构描述 + 风险提示 + 不确定性声明**

---

## 三、输入界面（Input UI）

### 输入字段

* 出生日期（年 / 月 / 日 / 时）
* 日历类型（公历 / 农历）
* 性别

### UI 提示文案（必须显示）

> 本系统采用节气换日、北京时间排盘规则
> 所有分析基于时间结构模型，不构成任何结果承诺

⚠️ 不允许用户：

* 自选节气规则
* 自选流派
* 自定义算法

这是**工程系统，不是配置玩具**。

---

## 四、计算触发按钮（极其关键）

### ❌ 禁止按钮文案

* “开始算命”
* “查看运势”
* “测一测你有多旺”
* “今年财运如何”

### ✅ 唯一允许方向

按钮文案示例（任选其一）：

* **生成时间结构分析**
* **计算时间结构热力图**
* **构建个人时间结构模型**

---

## 五、Heatmap 展示规则（核心）

### 1️⃣ Heatmap 的官方定义（UI 必须明示）

在 heatmap 区域顶部，必须固定显示：

> **颜色强度表示：该时间层级中某类结构被激活的相对强度**
> **不表示好坏，不表示结果概率**

---

### 2️⃣ 层级展示方式（强制）

UI 必须遵循：

```
年视图（默认）
 ↓ 点击
月视图
 ↓ 点击
日视图
 ↓ 点击
时视图
```

规则：

* ❌ 禁止一屏展示所有层级
* ❌ 禁止“全年运势总分”
* ✅ 只允许逐级下钻

---

### 3️⃣ Heatmap 颜色语义（必须统一）

* 冷色：结构低活跃 / 低暴露
* 暖色：结构高活跃 / 高暴露

⚠️ 禁止使用：

* “吉 / 凶”标签
* “好 / 坏”图例
* “红=好 / 绿=坏”这种情绪引导

---

## 六、十神展示方式（去玄学化）

### ❌ 禁止 UI 用语

* “你命里有财”
* “官运亨通”
* “印旺必贵”
* “伤官见官，必有灾”

### ✅ 统一替换为结构语言

| 十神 | UI 显示名（推荐） |
| -- | ---------- |
| 财  | 资源获取结构     |
| 官  | 约束 / 责任结构  |
| 印  | 支持 / 缓冲结构  |
| 食伤 | 输出 / 波动结构  |
| 比劫 | 竞争 / 内耗结构  |

---

## 七、行为提示区（最容易翻车的地方）

### 1️⃣ 行为提示区的定位

这是 **风险提示区**，不是建议区。

标题示例：

* **结构提示（非建议）**
* **当前时间结构下的风险说明**

---

### 2️⃣ 输出格式（强制）

所有行为提示必须符合：

> **“在当前时间结构下，某类行为的风险暴露水平为：高 / 中 / 低”**

示例：

* “当前时间结构下，高波动性行为风险暴露偏高”
* “长期承诺类决策结构承载力不足”
* “学习 / 内部修复类行为结构匹配度较高”

---

### ❌ 明确禁止

* “建议你去投资”
* “适合买彩票”
* “今年适合结婚 / 离婚”
* “你应该 / 你必须”

---

## 八、强制免责声明（UI 固定区）

页面底部或侧边，必须常驻显示：

> 本系统基于时间结构模型输出相对强度分析
> 不预测具体事件，不提供结果保证
> 所有信息仅用于结构理解与风险识别

不得折叠，不得隐藏。

---

## 九、交互上的“反人性设计”（必须有）

UI 必须做到：

* ❌ 不提供“一键结论”
* ❌ 不给“年度总评”
* ✅ 必须通过多次点击、下钻才能看到细节
* ✅ 让用户**为理解结构付出认知成本**

这是为了：

> **防止滥用，过滤不适合使用该系统的人**

---

## 十、最终 UI 铁律

> **如果一个 UI 设计：**
>
> * 让用户感觉“被安慰了”
> * 或感觉“被保证了”
>
> 那它一定是错误设计。

**UI 的职责不是取悦，而是约束误解。**
